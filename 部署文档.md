# 加盟商管理系统 部署文档

## 一、项目概述

本项目为加盟商管理总后台系统，包含后端 API 服务和前端管理界面，实现加盟商申请审核、合同管理、门店管理、保证金管理等核心功能。

- **后端技术栈**：Node.js + Express + Jest
- **前端技术栈**：Vue 3 + Vite + Element Plus
- **后端端口**：3060
- **前端端口**：5174

## 四、审核账号配置

### 4.1 审核人员账号

系统内置审核人员角色，审核操作时需指定审核人信息。

#### 4.1.1 默认审核账号

| 账号类型 | 默认值 | 说明 |
|----------|--------|------|
| 系统管理员 | `系统管理员` | 未指定审核人时的默认值，见 [franchiseService.js](file:///Users/wuzhijie/Documents/xiaohongshu/biaozhu/yxz-trae-projects/022-总后台加盟商门店管理/backend/src/franchiseService.js#L91-L91) |
| 前端默认审核人 | `管理员` | 前端界面默认使用的审核人，见 [ApplicationDetail.vue](file:///Users/wuzhijie/Documents/xiaohongshu/biaozhu/yxz-trae-projects/022-总后台加盟商门店管理/frontend/src/views/ApplicationDetail.vue#L581-L581) |

#### 4.1.2 账号权限配置

审核账号拥有以下操作权限：

1. **资质审核权限**
   - 通过/驳回加盟商申请
   - 填写审核意见
   - 驳回时必须填写驳回原因（不能为空或空白字符）
   - 代码位置：[franchiseService.js](file:///Users/wuzhijie/Documents/xiaohongshu/biaozhu/yxz-trae-projects/022-总后台加盟商门店管理/backend/src/franchiseService.js#L85-L87)

2. **阶段推进权限**
   - 资质审核 → 合同签约
   - 合同签约 → 加盟培训
   - 加盟培训 → 账号开通
   - 账号开通 → 上线运营

3. **门店账号管理权限**
   - 重置门店账号密码（密码长度不少于6位）
   - 启用/禁用门店账号
   - 代码位置：[storeService.js](file:///Users/wuzhijie/Documents/xiaohongshu/biaozhu/yxz-trae-projects/022-总后台加盟商门店管理/backend/src/storeService.js#L293-L295)

#### 4.1.3 审核接口说明

**审核申请接口**
```
PUT /api/applications/:id/audit

请求参数：
{
  "status": "approved" | "rejected",
  "auditOpinion": "审核意见",
  "auditor": "审核人姓名"
}
```

**审核规则**：
- 驳回时 `auditOpinion` 不能为空
- 已审核的申请不可重复审核
- 通过审核后自动推进到「资质审核」阶段（onboardingStage = 2）

## 五、合同附件配置

### 5.1 合同数据结构

合同信息包含以下核心字段，见 [contractService.js](file:///Users/wuzhijie/Documents/xiaohongshu/biaozhu/yxz-trae-projects/022-总后台加盟商门店管理/backend/src/contractService.js#L4-L99)：

| 字段 | 类型 | 说明 | 必填 |
|------|------|------|------|
| contractNo | String | 合同编号（HT前缀） | 自动生成 |
| partnerName | String | 合伙人姓名 | 是 |
| partnerPhone | String | 联系电话 | 是 |
| companyName | String | 公司名称 | 否 |
| storeName | String | 门店名称 | 是 |
| storeNo | String | 门店编号（MD前缀） | 是 |
| levelName | String | 加盟等级 | 是 |
| startDate | String | 合同开始日期 | 是 |
| endDate | String | 合同结束日期 | 是 |
| signDate | String | 签约日期 | 自动 |
| depositAmount | Number | 保证金金额（元） | 是 |
| serviceFeeRate | Number | 服务费率（%） | 是 |
| status | String | 合同状态 | 自动 |

### 5.2 加盟等级配置

系统支持以下加盟等级，见 [ApplicationDetail.vue](file:///Users/wuzhijie/Documents/xiaohongshu/biaozhu/yxz-trae-projects/022-总后台加盟商门店管理/frontend/src/views/ApplicationDetail.vue#L107-L111)：

| 等级 | 建议保证金 | 建议服务费率 |
|------|------------|--------------|
| 标准加盟商 | 50,000 元 | 5% |
| 银牌加盟商 | 100,000 元 | 4% |
| 金牌加盟商 | 200,000 元 | 3% |
| 钻石加盟商 | 500,000 元 | 2% |

### 5.3 合同状态流转

```
待签约 (pending)
    ↓
生效中 (active)  ← 签约确认
    ↓
已到期 (expired) ← 合同到期
    ↓
已终止 (terminated) ← 手动终止
```

### 5.4 合同附件存储配置

#### 5.4.1 附件上传目录配置

建议在后端项目根目录创建 `contracts` 目录用于存储合同附件：

```bash
mkdir -p backend/contracts/{templates,uploads}
```

目录结构：
- `contracts/templates/` - 存放合同模板文件
- `contracts/uploads/` - 存放已签署的合同扫描件

#### 5.4.2 合同模板配置

建议配置以下合同模板：

| 模板名称 | 适用等级 | 存放路径 |
|----------|----------|----------|
| 标准加盟商合同模板.docx | 标准加盟商 | contracts/templates/标准加盟商合同模板.docx |
| 银牌加盟商合同模板.docx | 银牌加盟商 | contracts/templates/银牌加盟商合同模板.docx |
| 金牌加盟商合同模板.docx | 金牌加盟商 | contracts/templates/金牌加盟商合同模板.docx |
| 钻石加盟商合同模板.docx | 钻石加盟商 | contracts/templates/钻石加盟商合同模板.docx |

#### 5.4.3 自动创建合同

当加盟商完成「账号开通」阶段推进到「上线运营」时，系统会自动创建合同：

1. 触发点：`advanceStage` 方法推进到阶段6时
2. 合同数据来源：`stageData.contract` 中的配置
3. 代码位置：[server.js](file:///Users/wuzhijie/Documents/xiaohongshu/biaozhu/yxz-trae-projects/022-总后台加盟商门店管理/backend/server.js#L98-L107)

## 六、部署步骤

### 6.1 后端部署

```bash
# 1. 进入后端目录
cd backend

# 2. 安装依赖
npm install

# 3. 启动服务（生产环境）
npm start

# 或开发环境
npm run dev
```

服务启动后访问：http://localhost:3060

### 6.2 前端部署

```bash
# 1. 进入前端目录
cd frontend

# 2. 安装依赖
npm install

# 3. 开发环境运行
npm run dev

# 4. 生产环境构建
npm run build

# 5. 预览构建结果
npm run preview
```

前端开发环境访问：http://localhost:5174

### 6.3 Nginx 配置示例（生产环境）

```nginx
server {
    listen 80;
    server_name franchise-admin.example.com;

    # 前端静态资源
    location / {
        root /path/to/frontend/dist;
        try_files $uri $uri/ /index.html;
        index index.html;
    }

    # API 代理
    location /api {
        proxy_pass http://127.0.0.1:3060;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    }
}
```

## 八、常见问题

### 8.1 端口被占用

```bash
# 查找占用端口的进程
lsof -i :3060
lsof -i :5174

# 杀死进程
kill -9 <PID>
```

### 8.2 跨域问题

前端已配置代理，见 [vite.config.js](file:///Users/wuzhijie/Documents/xiaohongshu/biaozhu/yxz-trae-projects/022-总后台加盟商门店管理/frontend/vite.config.js#L12-L19)：

- 生产环境：通过 Nginx 反向代理

### 8.3 测试数据重置

由于系统使用内存数据，重启后端服务后数据会自动重置为初始模拟数据。如需保留数据，请集成数据库（建议 MongoDB 或 MySQL）。
